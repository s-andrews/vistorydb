Combining and Reporting on Multiple Statistical Tests

For some types of analysis it makes sense to run multiple statistical tests and then select hits which are either the union or intersection of those hits. In earlier versions of SeqMonk there wasn't a nice way to then report on the final set of hits which preserved the interesting information from all of the tests, but now there is.

In this vistory we're going to look at reporting on hits from both DESeq and Intenstiy Difference tests on an RNA-Seq dataset, but the same technique can be used to report on any combination of list annotations.

I'm using a simple 2-condition, 3 replicate dataset to illustrate this. The data has already been imported and prepared.

Running the tests

I'm going to give a fairly terse run through the actual stats since those aren't the main point here. Other vistories go through the preparation of your data in more detail and you should look at those if you want to know more about actual RNA-Seq analysis. Here we're just focussing on the reporting.

DESeq

For the DESeq test we'll need to generate raw counts. In this dataset our library is opposing strand specific, but obviously you'd select whatever was appropriate for your data.

Probes Quantitated

RNA-Seq pipeline quantitation on merged transcripts counting reads over exons as raw counts. Assuming a Opposing strand specific library

Quantitation was RNA-Seq pipeline quantitation on merged transcripts counting reads over exons as raw counts. Assuming a Opposing strand specific library

Intensity Difference

For the ID test we're going to need log2RPM values, so we'll redo the quantitation to generate these normalised data.

Probes Quantitated

RNA-Seq pipeline quantitation on merged transcripts counting reads over exons. Log transformed. Assuming a Opposing strand specific library

New Probe List: Intensity Difference p<0.05 (95 probes)

[Intensity Difference Filter] Filter on probes in All Probes where minimum p-value when comparing KO to WT was below 0.05 multiple testing correction applied with a sample size of 261 when constructing the control distributions. Quantitation was RNA-Seq pipeline quantitation on merged transcripts counting reads over exons. Log transformed. Assuming a Opposing strand specific library

Combining the results

We now have two lists, one from each stats test, each of which has its own p-value, fdr and measure of difference. For our final list we only want to report on genes which are present in both lists, so let's combine them.

New Probe List: Significant by both DESeq and ID (95 probes)

[Combine Filter] DESeq stats p<0.05 after correction AND Intensity Difference p<0.05

If we take a quick look at the results then we can see that we have selected a resticted set of the most interesting hits.

We now want to report on these, but we want to have the information from both the DESeq and ID results. We also want to have the mean values for each condition so we can see the original log2RPM values as well.

Reporting

Selecting the data to report on

The report will include the current quantitation for whichever data stores are currently visible in the chromsome view. We want to see the the mean values per condition so I'll make the replicates sets visible and make sure that the "Expand replicate sets" option is turned off in the display preferences.

Generating an Annotated Probe Report

I'm finally going to generate an annotated probe report. The base preferences will be left as-is, but I'm going to select a custom list of annotation values using the option within the report options.

To add the annotation I select the probe list which contains them from the box on the top left, then I select the values I want to report from the top right, and press the button to add them.

The list at the bottom is the set of values which are going to end up in the final report. Here you can see I report the FDR and difference values for both the DESeq and ID tests. When I look back at my Annotated Probe List report options I can see that I now have 4 annotations selected (it was 0 before). I've also changed the options to show that I'm going to annotate each probe with the details of the gene with the same name.

This will now generate the complete report with all of the data I want.
