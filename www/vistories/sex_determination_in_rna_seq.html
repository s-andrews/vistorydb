Checking the sex of RNA-Seq samples

One common factor which gets overlooked in a large number of experiments is the sex of the subjects. Checking the sex of your samples should be a routine part of any analysis. Knowing that the sex of your samples matches what you expect is a key validation that you're looking at the right data. If you don't know the sex of your samples then finding this out can be important since this might be a confounding factor in your analysis.

In this example I'm going to take a set of 45 RNA-Seq samples and determine their sex. We won't go beyond that in this case, but in the context of a larger study we would match these up with the expectations from our metadata and would check against our experimental groups to make sure we hadn't confounded any of our other experimental factors with the sex of the samples.

I've already done a little bit of work. I quantitated all of the samples using the standard RNA-Seq pipeline to produce the default log2RPM values, and I did a post-quantitation normalisation using size-factor normalisation.

I also did a little bit of filtering to remove any gene which was unexpressed or very lowly expressed in all of my samples so they don't add to the noise.

Analysis

Filtering

We're going to make up some probe lists to make our life easier later on. We want to make the following lists.

1. Genes on the X Chromosome

2. Genes on the Y Chromosome

3. Genes on Autosomes (not X or Y, and if we have MT data we'd exclude that too - we don't here though)

4. Xist (the gene which causes X chromosome inactivation in females)

Xist expression

In mature samples the easiest way to tell males from females is to look at the expression of Xist. This is hightly expressed in females since it produces the non-coding RNA which coats the inactive X chromosome. In males it is virtually silent. We can use the Star Wars plot on the Xist list to see the values of this probe across all samples.

We can immediately see that we can divide the samples into two based on this metric alone. Samples with an Xist expression >=3 (in this case) should be female, and the rest should be male.

We can see that there is one sample (Sample13) which has an intermediate Xist expression. We can look at the raw data to try to work out why.

We can see that there is a bit of Xist expression in 13, but nowhere near the amount in the truly female samples. It looks likely that this is male, but maybe with a little bit of contamination. We can examine this more closely afterwards.

We next use a little trick to rename the male and female samples. From the Data Store Summary Report we order by Xist and then copy the names of the female samples. We then open the Data Set Editor and use the "Select named data sets" option to select the names we copied. We can then use the find/replace option to replace "Sample" with "Female Sample". We then repeat the process with the Males. We now have the names tagged with their gender.

X and Y Chr Expression

As a quick validation we can also now look at the mean expression for our identified Males and Females and map on the genes from the X and Y chromosomes, as well as Xist.

We can see that the X chromsome expression is a very poor predictor of sex due to X chromosome inactivation in females. Both Xist and the expression of genes on the Y chromosome are better predictors.

Y chromosome expression

If for some reason Xist expression was not informative, or indeed available, we can look at the Y chromsome expression. We can do this fairly simply based on the existing quantitation we have using our Y chromsome list and the star wars plot.

This works pretty well and you can see the difference between the autosomes and the genes on Y.

One issue we might have is that this measurement is based on the mean expression, so it can be quite skewed by a very highly expressed outlier. We can potentially get a more stable and relevant measure if we apply per-probe normalisation to the data so we're looking at the relative expression in each sample compared to the whole set.
