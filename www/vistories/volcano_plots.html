Assessing statistical filter results using Volcano Plots

When using statistical filters it is common to use a standard cutoff to decide which results should count as hits. Something like FDR<0.05 is common. Sometimes this filtering will be supplemnted with some kind of magnitude of effect filter such as an intensity difference statistical test, or a simpler fold change cutoff filter.

Rather than just having fixed cutoffs it can be useful to visualise the properties of a statistical test using a plot called a Volcano plot. This plots the p-value (or False Discovery Rater) against the absolute change (usually fold change). Looking at the results in a graphical way can help to determine whether the test has worked as expected, and whether there are sensible objective cutoffs to use for filtration.

Data

The Volcano Plot can be generated from any statistical filter available in SeqMonk. In this case we're using the DEeq2 filter on some raw RNA-Seq counts, but the same process would work for any test. In this instance the data has been imported and quantiated as raw counts using the RNA-Seq quantitation pipeline. We're going to do a simple 2-way comparison and view the results.

Running tests

Conventional filtering

In a conventional filter we set a fixed FDR cutoff. The resulting hit list will record the p-value, fdr and fold change, but only for the points which pass the FDR cutoff.

New Probe List: DESeq stats p<0.05 after correction (8022 probes)

DESeq2 stats filter on probes in All Probes where KO vs WT had a significance below 0.05 after Benjamimi and Hochberg correction. Quantitation was RNA-Seq pipeline quantitation on merged transcripts counting reads over exons as raw counts. Assuming a Opposing strand specific library

We can see that we get a very large number of hits, so we might use the ListAnnotationValue filter to apply an additional fold change filter to reduce the number of hits we have to deal with.

New Probe List: Absolute Log2 Fold Change above 2 (561 probes)

Filter on probes in DESeq stats p<0.05 after correction where annotation Log2 Fold Change from DESeq stats p<0.05 after correction had an absolute value above 2.0

Now we get a more managebale number of hits but we had to select the FDR and fold change cutoffs arbitrarily. We might have made different decisions if we could have seen the full set of hits from the original test.

Volcano Plot Filtering

When using volcano plots you want to see all of the results from a statistical filte, whether they are significant or not. To get these we need to change the filter options for the statistical test. Specifically we need to set the p-value cutoff to be 1 (the maximum value it can have), and make sure that multiple testing correction is not applied. In this way we will get a record for every test which was performed.

New Probe List: DESeq stats p<1.0 (18570 probes)

DESeq2 stats filter on probes in All Probes where KO vs WT had a significance below 1.0. Quantitation was RNA-Seq pipeline quantitation on merged transcripts counting reads over exons as raw counts. Assuming a Opposing strand specific library

You can see that not every probe is returned even with these settings. This is becase the statistical filter may opt not to test some points because the data is not valid for the test. In this instance DESeq2 will not test any point whose counts are zero in all conditions, so 18570 is the number of probes for which there was any data.

Now that we have the complete set of results we can draw the volcano plot. We must select the probe list we just created before drawing the plot. In the options for the plot we need to select an annotation value which represents difference, that would be fold change in this case. You also need to select the signficance measure you want to use. We could use either the p-value or the False Discovery Rate, but since we're going to ultimately filter on FDR we will use that.

We can now see the relationship between the FDR and fold change values. We can see that there is a set of related points spreading from the bottom end of the plot, mostly with negative fold changes. These might be points which are not specifically interesting, but are the outer edge of changes which are general to the overall disruption of the system. If we set our FDR cutoff low enough to avoid these then we might get a better set of hits. We can see that a value above 5 would avoid these, this equates to an FDR cutoff of < 10e-5. We can implement this with a list annotation value filter.

New Probe List: FDR below 1.0E-5 (4048 probes)

Filter on probes in DESeq stats p<1.0 where annotation FDR from DESeq stats p<1.0 had a value below 1.0E-5

Now we can see that there are a set of crowded points which are relatively close to the centre line for fold change. Again we might want to exclude these to get a better set of hits. Since we're just filtering on the output of the statistical test we have to use fold change for this. We can see that an absolute difference above 1 will cut out the majority of the unwanted points.

New Probe List: Absolute Log2 Fold Change above 1 (969 probes)

Filter on probes in FDR below 1.0E-5 where annotation Log2 Fold Change from DESeq stats p<1.0 had an absolute value above 1.0

This gives us a set of hits with cutoffs which we can see are going to be relatively specific for both of the metrics we used, and will therefore hopefully be more biologically relevant.

Comparing Results

Now we have generated two lists, one by more conventional filtering, and one using values determined from volcano plots we can look at them against the original datato see how they look. To do this we will need to use log transformed normaliseed expression values.

Probes Quantitated

RNA-Seq pipeline quantitation on merged transcripts counting reads over exons. Log transformed. Assuming a Opposing strand specific library

Conventional filtering

With conventional filtering we can see that there are a large number of hits in very low expressed genes, which have probably only generated high fold changes because one of the values was zero (or very close to it). It's likely that these points are not going to be the most biologically interesting.

Volcano Plot Filtering

The equivalent plot from the Volcano plot filtereing looks quite different. The low expressed, low difference changes are gone and more of the smaller changes in highly expressed genes remain, due to being able to set a more objective cutoff for the fold change. This set of hits would appear to be more biologically relevant.
