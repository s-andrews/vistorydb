Calling Peaks on ChIP data with Replicates

In this vistory I'm going to look at calling peaks on some replicated ChIP data. We have two replicates and a single input and we're going to look at a couple of options for how to build a peak list from them.

At the end we want to have a single set of probes which are the peaks we would take forward for further analysis.

At the start we've just imported the data and have done a simple running window quantitation. We can see that we have strongly enriched peaks in both of the replicates and that in general the peak positions look similar.

Strategies for multiple peak calling

We are going to use two different strategies to call peaks.

1. We can make a merged set of the ChIP samples and call a single set of peaks on this. This is a simple approach but will have some limitations if there are places where the peak enrichments are not similar between the two samples.

2. We can call peaks separately on the two datasets and then combined the hits in one of the number of ways to get to a final called set. This can give us a more selectively called set, but might have problems if the quality of both replicates is not similar.

Creating a blacklist

One issue which will potentially cause us trouble in our analysis is badly behaving parts of the genome. To make life easier later on we are going to create a blacklist of regions on the genome which have unusual coverage in the input sample so that we can ignore peaks which fall into these regions later on. These regions usually represent mis-mapped or unmappable parts of the genome so they're not generally providing useful biological information anyway.

We have quantitated our data with a running quantitation so we can look at the histogram of quantitated values in our input.

We can see that there are some very highly covered outliers which skew the view of the overall disitrubution of values. We can zoom in on the bottom end of the distributiuon of values to get a better idea of what's going on in the nicely behaving data.

We can see a spike at zero counts which are holes in the assembly or unmappable regions. We can also see that by a count of 80 in the input that all of the normally behaving parts of the genome are finished with. We can therefore apply a filter to get the nicely behaving parts of the genome, and also the badly behaving bits.

Calling Peaks

Merged Data

The first (and probably easiest) option is to call peaks based on merging together the raw data from all of the samples. This has the big advantage that you only have to do one set of calling. It also means that by combining the data from the replicates we maximise our statisitcal power to get the most possible peak calls. It does have some assumptions;

1. That you have replicates with reasonably similar coverage and enrichment

2. That you're only calling replicates in one or two conditions, and that the number of replicates per condition is similar.

If these conditions are met (and they are in our case) then we can go ahead. We can simply select both replicates when we run our peak call.

We'll use the feature probe generator to generate an initial combined set of peaks, and then the deduplication generator to merge them.
